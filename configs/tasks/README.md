# Сид-очередь исследовательских задач

Оркестратор грузит все `*.yaml` при старте (идемпотентно по `id`). Читаются
`id`, `kind` (exact|maximize|open), `statement`, `cap_usd`. Прочие поля —
комментарии. Новые задачи добавляются на лету: боту свободным текстом.

Домен — program search / алгоритмическое открытие: CPU-only, результат объективно
проверяем арбитром (валидность инварианта + сравнение с бейзлайном/границей).

| id | cap | суть |
|---|---|---|
| cap_set_construction | $4 | макс. cap set в F_3^n, n=4..7 (флагман FunSearch) |
| online_bin_packing_heuristic | $3 | эвристика онлайн bin packing лучше best/first-fit |
| sorting_network_min_comparators | $3 | сортирующие сети с мин. компараторами, n=9..12 |
| largest_no_3ap_set | $2 | макс. подмножество {0..N-1} без 3-членной АП |
| maxcut_local_search | $2 | быстрая локальная эвристика MAX-CUT на бенчмарке |
