# Стартовая очередь исследовательских задач

Оркестратор при старте грузит все `*.yaml` отсюда (идемпотентно по полю `id`).
Читаются `id`, `kind`, `statement`, `cap_usd`; `verification`/`budget_rationale` —
для человека. Новые задачи можно и не класть сюда, а добавлять на лету через бота
(`/addtask`).

Все задачи домена **program search / алгоритмическое открытие**: вычислительно
лёгкие (без GPU), результат объективно проверяем арбитром (валидность инварианта +
сравнение с бейзлайном/известной границей), пересекаются с competitive programming.

| Задача | kind | cap | суть | проверка |
|---|---|---|---|---|
| cap_set_construction | maximize | $15 | максимальный cap set в F_3^n, n=4..7 (флагман FunSearch) | инвариант «нет тройки суммой 0» + размер vs greedy/границы |
| online_bin_packing_heuristic | maximize | $12 | эвристика онлайн bin packing лучше best/first-fit | реплей сидов, нет переполнения, среднее число корзин |
| sorting_network_min_comparators | maximize | $12 | сортирующие сети с мин. компараторами, n=9..12 | 0-1 принцип (все 2^n входов) + счёт компараторов |
| largest_no_3ap_set | maximize | $10 | макс. подмножество без 3-членной АП, N=256..2048 | проверка инварианта + \|S\| vs greedy |
| maxcut_local_search | maximize | $8 | быстрая локальная эвристика MAX-CUT на бенчмарке | реплей графов, значение реза, тайм-бюджет vs бейзлайны |

## Раскладка бюджета

Сумма cap'ов одного полного прохода: **$57** (это потолки, не факт) — против общего
лимита **$250** это ~23% в худшем случае. С механикой конкуренции реальные cap'ы у
агентов ниже качества-1.0 масштабируются вниз (пол `min_share=0.15`), так что
фактический расход на цикл — обычно **единицы–первые десятки долларов**. Плюс accrual
сервера ($30-50/мес). При $250 бюджета сервер съедает его за ~5-8 месяцев простоя,
поэтому следи за accrual — на дашборде и в `/budget`.

Порядок сложности возрастающий намеренно: `maxcut` (дешёвый) и `no_3ap` калибруют
качество агентов до того, как коллегия тратит на более дорогой открытый поиск
(`cap_set`). budget-guard в любом случае не даст превысить ни per-task cap, ни
общий лимит.
