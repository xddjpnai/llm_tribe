# Стартовая очередь исследовательских задач

Оркестратор при старте грузит все `*.yaml` отсюда (идемпотентно по полю `id`).
Читаются `id`, `kind`, `statement`, `cap_usd`; `verification`/`budget_rationale` —
для человека. Новые задачи можно и не класть сюда, а добавлять на лету через бота
(`/addtask`).

Все задачи домена **program search / алгоритмическое открытие**: вычислительно
лёгкие (без GPU), результат объективно проверяем арбитром (валидность инварианта +
сравнение с бейзлайном/известной границей), пересекаются с competitive programming.

| Задача | kind | cap | суть | проверка |
|---|---|---|---|---|
| cap_set_construction | maximize | $4 | максимальный cap set в F_3^n, n=4..7 (флагман FunSearch) | инвариант «нет тройки суммой 0» + размер vs greedy/границы |
| online_bin_packing_heuristic | maximize | $3 | эвристика онлайн bin packing лучше best/first-fit | реплей сидов, нет переполнения, среднее число корзин |
| sorting_network_min_comparators | maximize | $3 | сортирующие сети с мин. компараторами, n=9..12 | 0-1 принцип (все 2^n входов) + счёт компараторов |
| largest_no_3ap_set | maximize | $2 | макс. подмножество без 3-членной АП, N=256..2048 | проверка инварианта + \|S\| vs greedy |
| maxcut_local_search | maximize | $2 | быстрая локальная эвристика MAX-CUT на бенчмарке | реплей графов, значение реза, тайм-бюджет vs бейзлайны |

## Раскладка бюджета

Капы намеренно скромные («не тратить много»). Сумма cap'ов одного полного прохода:
**$14** (это потолки, не факт) — против страховочного потолка **$100** это ~14%.
С механикой конкуренции реальные cap'ы у агентов ниже качества-1.0 масштабируются
вниз (пол `min_share=0.15`), так что фактический LLM-расход на цикл — обычно
**единицы долларов**. Отдельно идёт accrual сервера (~$40/мес); при $100 потолка
именно он — главный расход простоя, следи за ним в `/budget` и на дашборде.

Потолок $100 — не цель, а «стоп + пинг тебе». Хочешь дать коллегии больше — подними
`total_budget_usd` в `configs/budget.yaml` (или `BUDGET_TOTAL_USD` в кредах) и
перезапусти budget-guard. Пополнение самих API — на тебе.

Порядок сложности возрастающий намеренно: `maxcut` (дешёвый) и `no_3ap` калибруют
качество агентов до того, как коллегия тратит на более дорогой открытый поиск
(`cap_set`). budget-guard в любом случае не даст превысить ни per-task cap, ни
общий потолок.
